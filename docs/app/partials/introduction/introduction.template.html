What is Blueprints?
An UI framework using Bootstrap 3 as its foundation. Created in that way primarily focusing on look and feel, described as a long living styleguide that can evolve over time and empower developers to build application pages quickly whilst conforming with a consistent branding that is agreed and defined.
Start using this styleguide
You have several options to include the distribution files into your project but the result should be the same: get a copy of the stylesheet file and the javascript into your main HTML document.

There are two recommended options:
  • Option 1: Getting a copy of the distribution files
    Make a copy of the compiled files contained in the distribution folder into your project
    (The following block shows the code you need to include in your index.html but don't forget to include to make acopy of the dist/ folder and all its content).
    <link rel="stylesheet" type="text/css" href="dist/css/blueprints.css"> <script type="text/javascript" src="dist/js/blueprints.js"></script>
  • Option 2: Adding Blueprints as a node module (can specify version)
    You will include the "compiled" files into your project, you may have a grunt/gulp task to handle those. Of course you use the HTML code above but with this approach you are aiming to have more control over those files, the following is an example of your package.json file
    { //Your HTML code will looks similiar to: //<link rel="stylesheet" type="text/css" href="node_modules/dist/css/blueprints.css"> //<script type="text/javascript" src="node_modules/dist/js/blueprints.js"></script> "dependencies": "git+ssh://git@github.private.linksynergy.com/SD/blueprints.git#dev" }
