Badges
Eye candy element. Regular HTML markup element

You can add this element to practically all block elements available; the main goal of this element is to easily highlight with (or without) a number any new or unread items. Think about this as a variation of a notification you can add to your elements.

— How to use this element?

There are two types of badges you can take advantage of: block and overlapped badges
You can only use one of the following badge-success, badge-warning or badge-danger

button.btn-default 4
The element tag represents any block element you want to use. That element may or may not have already attached classnames {...}, yet you only need to add badge classname to that list. ¹{classnames} is optional but besides including one of the classnames described above ↑; you can add extra modifier: see the following table for a detailed explanation; if not modifier present the badge will use the default text colors of the body itself

Considerations: When the element used to show the badge element doesn't include any Textnode value, the element will try to collapse itself, via CSS's :empty selector. Bear in mind this functionality will not work for Internet Explorer 8 and below since its lack of support for such selector.
Classname Description
badge-before If present; this will put the badge before the label; moving it to the left instead of using the default right position.
badge-square By default the badge will be displayed in a circle; adding this classname will modify it to be a rectangle
