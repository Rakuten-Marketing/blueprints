What is Blueprints?

You could think of Blueprints as an UI framework, primarily focused on look and feel. It's a long living styleguide that can evolve over time and empower developers to build application pages quickly whilst conforming with a consistent branding that is agreed and defined.

Essentially built on top of Bootstrap 3.

How can I include Blueprints in my project?

There are two recommended options:

Using CDN or getting a copy of the distribution files into your project

Make a copy of the compiled files contained in the distribution folder into your project
NOTE: The following block shows the code you need to include in your index.html but don't forget to include a copy of the folder and all its content.

<link rel="stylesheet" type="text/css" href="dist/css/blueprints.css"> <script type="text/javascript" src="dist/js/blueprints.js"></script>

Installing Blueprints using NPM (You need a specific version of Blueprints)

You will include the "compiled" files into your project, you may have a grunt/gulp task to handle those. Of course you use the HTML code above but with this approach you are aiming to have more control over those files, the following is an example of your package.json file

{ //Your HTML code will looks similiar to: //<link rel="stylesheet" type="text/css" href="node_modules/dist/css/blueprints.css"> //<script type="text/javascript" src="node_modules/dist/js/blueprints.js"></script> "dependencies": "git+ssh://git@github.private.linksynergy.com/SD/blueprints.git#dev" }

NOTE: You can specify the branch (#dev in this example), this is good for those scenarios you need a specific version of Blueprints
