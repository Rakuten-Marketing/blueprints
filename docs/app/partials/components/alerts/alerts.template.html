Alerts
Responsive element/layout markup with a webcomponent option

Use this element to provide a contextual feedback message for typical user actions; these elements however are considered as block elements; meaning they are not floting alerts.

— How to use this element?

For all the variations of this element you only ned to use the alert classname followed by the {classnames} desired modifiers. Use the following code block to understand this:

Well done! You successfully read this important alert message.
Warning! Better check yourself, you're not looking too good.
Oh snap! Change a few things up and try submitting again.
Alerts don't have default classname, since a default gray alert doesn't make too much sense, so you're required to specify a type via contextual class. Choose from alert-success, alert-warning or alert-danger.
NOTE: please don't including the role="alert" attribute.

— Interactive elements inside alerts

The only requirement is adding the alert-interactive classname to any clickable element that is NOT a button (because the buttons have their own rules). Adding this classname will force the interactive elements to get the contextual color of the current alert. Use the following block to understand this:

This is an example showing a link
You can also include a button to this element, you will need to do things by yourself to achieve the desired layout; remember this element is just a block element
