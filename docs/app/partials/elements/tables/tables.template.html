Tables
Responsive element/layout markup with a webcomponent

If you want to have the zebra-striping and responsive behavior you can simply use the .table-default classname; which represents the default style for all our tables accross applications. Yet you can ignore that classname and assemble something ad hoc your needs.

Before we get started, bear in mind the following
The responsive behavior (.table-responsive) will require you to add an extra attribute to your td (tbody).

<td data-header="Header here"></td>

— How to use it?

Use this version for basic styling of your tables.
Regarding the caption: this element is optional but if you decide to include it, bear in mind this element is using the display: table-caption. This means, the caption will be always forced to be at the top (first child) of the table, because its nature. If you rather having something after the table and not before, don't include the caption element inside <table class="table {classnames}></table>" but after it.

— Table variations

Use the following adding the classnames to the table
<table class="table desired classnames></table>

Classname Description
table-default Rakuten Marketing's default look and feel for all the tables.
Bear in mind: includes .table-responsive and .table-striped behaviors.
table-responsive Not including this classname (doesn't apply if you are using .table-default) will affect responsiveness for all the elements in the active document. This because the table content has a limit regarding its horizontall compression. That width will be the new document's width forcing the scrolling functionality/need

This classname has also a subclassname .table-responsive-columns, this will allow you to show the tables as rows for mobile devices.
table-bordered This will add borders to all the rows in your table, by default you will see a very light-colored border but adding this classname will highlight them even more.
table-striped Adding the zebra-striping behavior to the rows. It is nice to have some kind of visual rest to denote boundaries between each row, for that reason the .table-default includes this. Striped tables are styled via the :nth-child CSS selector, which is not available in Internet Explorer 8.
table-hoverable You need to add this classname (not included in the .table-default behavior or anywhere really) to add hoverable actions to the table rows (body).
table-condensed This classnames will make you table thinner, playing with its padding values
table-simple Alternative look and feel for your tables. See this table explaining the classname usage? You got an example

— Classes you can add to the tr

You can only hide or disable these elements for now

— Classes you can add to the th and td

If you need to add new ones, please follow this pattern th.classname,td.classname {}
— Don't use text-align properties, inside your custom classnames
— Maybe you want to play with the width of your columns

Classname Description
preferred Will try to use the 100% available of the current table
restricted Deactivate the cell, blocking user interaction
number Formatting cells containing numbers
text Formatting cells containing text
date Formatting cells containing a date
actions Formatting cells containing interactive elements (buttons)
How to handle text alignment? Just add the helper classnames text-left, text-center, text-right, text-justify or text-nowrap
